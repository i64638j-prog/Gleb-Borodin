Краткое описание
GitHub User Finder — простое GUI-приложение на Python (Tkinter) для поиска пользователей GitHub, просмотра их профилей в браузере и сохранения избранных пользователей в локальный JSON-файл (favorites.json). Приложение использует GitHub Search API (/search/users).

Основные функции:

Поиск пользователей GitHub по запросу.
Отображение результатов в списке (login — html_url).
Открытие профиля пользователя в браузере (двойной клик).
Добавление пользователя в избранное.
Просмотр и удаление пользователей из избранного.
Сохранение списка избранных в JSON.
Установка и запуск
Клонируйте репозиторий или поместите файлы проекта в папку.
(Опционально) Создайте и активируйте виртуальное окружение:
Linux/macOS:
python3 -m venv venv
source venv/bin/activate
Windows (PowerShell):
python -m venv venv
.\venv\Scripts\Activate.ps1
Установите зависимость:
pip install requests
(Если в репозитории есть requirements.txt: pip install -r requirements.txt)
Запустите приложение:
python main.py
Файл favorites.json создаётся автоматически при первом добавлении в избранное (если отсутствует).

Как использовать GUI
Введите поисковый запрос в поле "Поиск пользователя GitHub".
Поле не должно быть пустым — при пустом вводе появится предупреждение.
Нажмите кнопку "Поиск" или нажмите Enter.
В результатах отобразятся найденные пользователи в формате "login — html_url".
Двойной клик по строке откроет профиль пользователя в веб-браузере.
Чтобы добавить пользователя в избранное:
Выделите строку в результатах и нажмите "Добавить в избранное".
Просмотр/удаление избранных:
Нажмите "Просмотр избранных", в новом окне можно открыть профиль или удалить запись.
Использование GitHub API
Приложение использует GitHub Search API для поиска пользователей.

Endpoint:

GET https://api.github.com/search/users
Параметры:

q — строка поиска (обязательный). Пример: "torvalds", "location:Russia followers:>100".
per_page — число результатов на страницу (опционально, можно до 100).
Пример запроса (curl):

Без авторизации:
curl "https://api.github.com/search/users?q=torvalds&per_page=5"
С использованием персонального токена (увеличивает rate limit):
curl -H "Authorization: token YOUR_TOKEN" "https://api.github.com/search/users?q=torvalds&per_page=5"
Фрагмент ответа (упрощённый):
{
"total_count": 1,
"items": [
{
"login": "torvalds",
"id": 1024025,
"avatar_url": "https://avatars.githubusercontent.com/u/1024025?v=4",
"html_url": "https://github.com/torvalds",
...
},
...
]
}

Важные замечания:

Неавторизованные запросы ограничены лимитом (обычно 60 запросов в час). При превышении лимита API вернёт 403. Для интенсивной работы используйте персональный токен GitHub и добавляйте заголовок Authorization.
Для аутентификации через токен добавьте заголовок: Authorization: token YOUR_TOKEN.
Примеры использования / тесты
Поиск конкретного пользователя:
Введите "torvalds" — вы должны увидеть запись с login = torvalds и html_url = https://github.com/torvalds.
Пустой поиск:
Нажмите "Поиск" с пустым полем — появится предупреждение "Поле поиска не должно быть пустым".
Добавление в избранное:
Найдите пользователя, выделите его в списке и нажмите "Добавить в избранное". Откройте "Просмотр избранных" — запись должна присутствовать.
Удаление из избранного:
В окне избранных выберите запись и нажмите "Удалить из избранного" — запись удалится и favorites.json обновится.
Просмотр профиля:
Двойной клик по результату или в окне избранных по записи — профиль откроется в браузере.
Проверка файла favorites.json:
После добавления в избранное откройте favorites.json — он содержит массив объектов вида:
[
{"login": "example", "html_url": "https://github.com/example"},
...
]

Замечания и улучшения
Текущее приложение делает неавторизованные запросы. Для повышения лимита запросов можно реализовать ввод персонального токена в настройках.
Можно добавить отображение аватаров, дополнительной информации о пользователе (followers, repos) и пагинацию.
Для больших объёмов запросов рекомендуется добавить обработку rate limit и сообщение пользователю о необходимости аутентификации.
